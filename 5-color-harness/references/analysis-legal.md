# 분석·법무

이 카테고리는 결함이 곧 손실 또는 분쟁으로 이어지는 모든 분석·법무 산출물을 다룬다. 엑셀 모델·시장 분석은 9.5군이지만, 계약서 검토·법률 의견서·송무 서면·감사 의견서·세무 의견서·실사 보고서는 모두 9.7군이다. 공통점은 (1) 인용·출처가 본문의 신뢰를 결정한다, (2) 모호 표현 한 줄이 책임 분쟁의 단초가 된다, (3) 가정·전제를 명시 안 하면 의견 자체가 무효화된다.

## 매핑 표

각 행은 RED=WHAT, SILVER=HOW, BLUE=FORM 카테고리에서 한 개씩 뽑혀 있다. 합격선이 표기되지 않으면 9.5다.

| 과제 | RED (WHAT) | SILVER (HOW) | BLUE (FORM) | 합격선 |
|------|-----|--------|------|--------|
| 엑셀 모델 | 가정 타당성 | 구조, 추적가능성 | 검증, 에러 방지 | 9.5 |
| 시장 분석 | 통찰 | 데이터 견고함 | 시각화 | 9.5 |
| 계약서 검토 | 리스크, 책임 분배 | 정합성, 모호성 | 실무 운영성 | **9.7** |
| 법률 의견서 (리걸 메모) | 법적 결론·리스크 등급 | 사실관계 → 법리 → 결론 구조 | 인용·전제·모호 표현 차단 | **9.7** |
| 소장·답변서 | 청구 원인·항변 사유 | 시계열·증거 정렬 | 형식·송달·당사자 표시 | **9.7** |
| 준비서면 | 쟁점별 반박·법리 | 상대 주장 → 법리 → 결론 | 인용 정합·간결 | **9.7** |
| 감사 의견서 (감사 보고서) | 적정·한정·부적정 결론 | 의견·한정 사유 구조 | 회계기준 인용 정합성 | **9.7** |
| 세무 의견서 (세무 메모) | 세무 결론·리스크 등급 | 법령·예규·판례 인용 구조 | 시행령·시행규칙 출처, 모호 표현 차단 | **9.7** |
| 실사 보고서 (Due Diligence) | 발견 사항·리스크 우선순위 | 카테고리별 구조 (재무·법무·운영·세무·인사) | 출처·후속조치·첨부 정합성 | **9.7** |
| 내부통제 평가 보고서 (Management Letter) | 통제 결함·개선 권고 | 결함·영향·권고 구조 | 등급·우선순위 표기 | 9.5 |

## 합격선 기준 장면

- 엑셀 모델. "다른 사람이 30분 안에 가정·계산·출력 흐름을 따라가고 셀 한 개를 바꿔도 결과가 깨지지 않는다."
- 시장 분석. "독자가 통찰 한 줄을 자기 의사결정에 그대로 옮긴다."
- 계약서 검토 (9.7). "사인 직전 한 조항에서 한 번 더 멈춰 묻고 싶어진다."
- 법률 의견서 (9.7). "클라이언트가 결론 한 줄을 보고 한 번 더 읽지 않고 의사결정한다."
- 소장·답변서 (9.7). "재판부가 첫 변론기일에 청구취지·청구원인을 한 번에 파악하고 보충 자료를 따로 청하지 않는다."
- 준비서면 (9.7). "재판부가 다음 기일에서 우리 측 쟁점 정리를 그대로 인용한다."
- 감사 의견서 (9.7). "감독 당국이 한정 사유 단락을 그대로 인용 가능하다."
- 세무 의견서 (9.7). "납세자가 결론 한 줄로 신고를 진행한다."
- 실사 보고서 (9.7). "ES 한 페이지만 보고 딜 진행 여부 1차 판단이 가능하다."
- 내부통제 평가. "감사위원회가 결함 표 한 장으로 우선순위를 정한다."

## 문체·금지 뱅크

### 엑셀 모델
- 문체. 가정 셀과 수식 셀을 색상으로 분리. 셀 코멘트로 출처 남기기. 한 시트 한 목적.
- 분량. 시트 5~10개. 가정 시트 1, 입력 시트 2~3, 계산 시트 2~4, 출력 시트 1~2.
- 금지. 가정 시트 밖의 하드코딩. 명명 없는 시트. 검증 시트 부재.

### 시장 분석
- 문체. 통찰 한 줄을 헤드에. 데이터 출처 명시. 비교군과 함께 보기.
- 분량. 3~10페이지, 5000~15000자. 차트 5~10개.
- 금지. 데이터 나열로 끝남. 출처 누락. 통찰 없는 요약.

### 계약서 검토 — 9.7
- 문체. 짧고 단정적. 정의 조항으로 모호 표현 차단하기. 책임 한도와 면책의 양방향성 명시.
- 분량. 검토 의견서 2~5페이지, 3000~7000자. 조항별 의견은 한 조항당 100~300자.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - "고려할 수 있다" 직접 인용 금지.
  - "합리적 노력" 직접 인용 금지.
  - "통상의 관행" 직접 인용 금지.
  - 진술과 보증의 누락 묵과.
  - 책임 한도 일방성 묵과.

### 법률 의견서 (리걸 메모) — 9.7
- 문체. 결론 한 줄을 머리에. 사실관계 한 단락. 법리 검토 본문. 결론 재서술. 각주에 출처. 가정·전제 명시.
- 분량. 5~15페이지, 5000~15000자. 짧은 사안은 2~3페이지.
- 금지 (9.7군).
  - "사료된다" 직접 인용 금지.
  - "보아야 할 것이다" 직접 인용 금지.
  - "해석상 다툼의 여지" 직접 인용 금지.
  - 가정 누락. 적용 시점 부재.
  - "추가 검토가 필요"로 끝맺기.

### 소장·답변서 (9.7군)
- 문체. 소장은 청구취지·청구원인 두 단락. 답변서는 청구취지에 대한 답변·청구원인에 대한 답변·항변 세 단락. 시계열·증거 일치. 당사자 표시 정확.
- 분량. 소장 10~30페이지. 답변서 5~20페이지. 단순 사안은 5~10페이지.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - "심히 부당" 직접 인용 금지.
  - "악의적" 직접 인용 금지.
  - 입증 없는 단정.
  - 형식 위반(송달 주소·당사자 표시 누락).
  - 청구취지가 모호한 추상 표현(특정 부재).

### 준비서면 (9.7군)
- 문체. 쟁점 정리 한 단락을 머리에. 상대방 주장 인용 → 우리 측 반박(법리·증거) → 소결의 3단을 쟁점마다 반복. 새 쟁점은 별도 단락 또는 별지.
- 분량. 15~50페이지. 단순 쟁점은 5~10페이지.
- 금지 (9.7군).
  - "사회 통념상" 직접 인용 금지.
  - "이미 법정에서 충분히" 직접 인용 금지.
  - 같은 주장을 다음 회차 준비서면에 재반복.
  - 상대방 주장 인용 부정확.
  - 쟁점 누락(상대 주장에 대한 무대응).

### 감사 의견서 (감사 보고서) — 9.7
- 문체. 적정·한정·부적정·의견거절 중 하나 명시. 한정·부적정이면 사유 단락 필수. 회계기준 인용. 비교 시점 일치.
- 분량. 본문 1~2페이지(의견서), 부속서 별도. 핵심감사사항(KAM)은 항목당 300~500자.
- 금지 (9.7군).
  - "전반적으로 적절" 직접 인용 금지.
  - "특별한 문제 없음" 직접 인용 금지.
  - 한정 사유 부재.
  - 비교 시점 불일치.
  - 회계기준 인용 누락.

### 세무 의견서 (세무 메모) — 9.7
- 문체. 결론 한 줄 머리에. 세법·예규·판례 인용 본문. 리스크 등급(낮음·중간·높음) 표시. 결론 재서술.
- 분량. 3~10페이지, 3000~10000자.
- 금지 (9.7군).
  - "유사 사례 기준" 직접 인용 금지.
  - "관행에 따라" 직접 인용 금지.
  - "통상" 직접 인용 금지.
  - 시행령·시행규칙 출처 누락.
  - 적용 시점·납세자 식별 부재.

### 실사 보고서 (Due Diligence) — 9.7
- 문체. Executive Summary 한 페이지를 머리에. 카테고리별 발견 사항(재무·법무·운영·세무·인사). 리스크 우선순위 표(High·Medium·Low).
- 분량. 30~100페이지. ES는 1~2페이지. 각 카테고리 5~15페이지.
- 금지 (9.7군).
  - "검토가 필요해 보임" 직접 인용 금지.
  - "향후 확인" 직접 인용 금지.
  - 출처 없는 발견.
  - 리스크 등급 부재.
  - 후속 조치 권고 부재.

### 내부통제 평가 보고서 (Management Letter)
- 문체. 결함 한 줄, 영향 한 줄, 권고 한 줄의 세 칸 표. 우선순위 등급. 책임자·기한 명시.
- 분량. 5~20페이지. 결함 항목당 200~400자.
- 금지. 등급 부재. 권고가 모호("개선 필요"). 책임자·기한 누락.

## fallback 5단계 합성 알고리즘

1. **결과물의 책임 무게를 본다.** 의견(클라이언트 의사결정 근거) / 보고(외부 공시) / 검토(내부 참고) / 모델(반복 사용) 중 무엇인가.
2. **인용 강도를 본다.** 법령·기준 직접 인용 필수 / 출처 표기만 / 자체 분석 중 무엇인가.
3. **두 후보를 고른다.** 책임 무게 일치 + 인용 강도 일치가 1순위.
4. **문체 합성.** 결론 한 줄 + 사실·법리·결론 또는 가정·계산·출력의 3단 + 출처 명시.
5. **금지 합성.** 모호 표현("고려할 수 있다·합리적 노력") + 회피 어미("추가 검토가 필요") 두 줄을 기본 금지로. 9.7군이면 그 영역 흔한 실패 문구 직접 인용 두 줄 추가.

예. **"보안 심사 보고서"**. 책임 무게 의견(공식 심사 결과). 인용 강도 기준(ISO·NIST) 직접 인용. 후보 = 감사 의견서 + 내부통제 평가 보고서. 합성. 적합·부적합 결론 한 줄 + 통제 항목별 결함·영향·권고 표 + ISO/NIST 인용. 9.7군 처리. "전반적으로 양호" 직접 인용 금지 + 등급 부재 금지.
