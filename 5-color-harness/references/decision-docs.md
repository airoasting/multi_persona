# 의사결정 문서

이 카테고리는 결정자(임원·이사회·주주·LP·투자위원회)가 한 번 보고 결정을 내리도록 만드는 모든 문서를 다룬다. 보고서·이사회 자료·주주 서한·IR·OKR 리포트·투자 메모·DD·산업 리포트 등이 모두 여기 속한다. 공통점은 (1) 결정자의 시간이 비싸다, (2) 첫 페이지·첫 줄에 결정 요청 또는 메시지가 와야 한다, (3) 숫자에 출처가 붙어야 한다, (4) 약속의 인플레이션을 피한다.

## 매핑 표

각 행은 RED=WHAT, SILVER=HOW, BLUE=FORM 카테고리에서 한 개씩 뽑혀 있다. 합격선이 표기되지 않으면 9.5다.

## 합격선 기준 장면

- 보고서 (1페이지). "임원이 한 장으로 결정을 내리고 추가 자료를 안 청한다."
- 보고서 (분석). "독자가 통찰 한 줄을 다른 회의에 그대로 옮겨 인용한다."
- 내러티브 메모. "리더가 첫 30분 침묵 독해 후 첫 질문이 결론 한 줄에 정확히 꽂힌다."
- 기획안·제안서. "의사결정자가 한 줄 요약을 보고 다음 검토 회의를 잡고 일정·비용 페이지로 직행한다."
- 영업 제안서·RFP. "구매 담당자가 첫 3페이지에서 점수를 매기고 인터뷰 short-list 후보로 올린다."
- 회의록. "다음 회의 직전 5분에 결정 사항만 한 번 훑고 액션 아이템 담당자·시점을 그대로 따라간다."
- 이사회 보고서 (9.7). "이사회가 첫 페이지에서 결정 요청을 정확히 알아채고 본문은 보충 자료로만 본다."
- 주주 서한 (9.7). "주주가 한 분기 더 가져갈 신뢰를 결말 한 단락에서 확인한다."
- PRD. "엔지니어가 한 번 읽고 user story를 카드로 끊고 P0/P1/P2 우선순위를 그대로 가져간다."
- IR·실적 발표 (9.7). "애널리스트 노트에 헤드라인 한 줄과 가이던스가 함께 인용된다."
- OKR 리포트. "다음 분기 OKR을 잡는 회의가 격차 원인 한 줄에서 시작한다."
- MD&A. "감독 당국이 변동 원인 한 줄을 그대로 인용 가능하다."
- 투자 메모 (9.7). "IC가 한 번 토의로 결정을 내리고 베이스·업·다운 케이스를 추가 질문 없이 받아들인다."
- LP 리포트. "LP가 다음 펀드 약정에 그대로 들어온다."
- 비즈니스 DD 메모. "딜 매니저가 가설 검증 표 한 장으로 다음 단계(Go·Hold·Pass)를 결정한다."
- 산업 리포트. "독자가 드라이버 한 줄을 자기 의사결정에 적용한다."
- 전문가 콜 노트. "투자 매니저가 핵심 인용 박스 두 줄을 IC 메모에 그대로 옮겨 적는다."
- 결론. "한 페이지 결론만 보고 본 덱 30장의 흐름을 예측한다."
- 진단 보고서. "클라이언트 임원이 ES 한 페이지로 컨설팅 권고 진행 여부와 다음 워크샵 일정을 결정한다."

## 문체·금지 뱅크

### 보고서 (실무, 1페이지)
- 문체. 액션 타이틀로 시작. So what을 끝까지 견지. 한 페이지 한 메시지.
- 분량. A4 한 장, 1500~2000자.
- 금지. "검토가 필요합니다"로 마무리. "배경부터 말씀드리면"식 도입. 결론과 데이터 사이의 한 단계 비약.

### 보고서 (분석)
- 문체. 통찰 한 줄을 헤드에. 데이터로 뒷받침. 반대 케이스 한 줄.
- 분량. 3~10페이지, 5000~15000자.
- 금지. 데이터 나열로 끝남. 통찰이 부재한 요약. "추가 분석이 필요"로 끝맺기.

### 내러티브 메모 (6-pager)
- 문체. 결론 한 문장을 첫 단락에. 본문은 완결된 문단으로 결론를 지지(불릿 금지). 각 단락이 한 결론를 받친다. 표·차트는 부록 별지로 분리. FAQ가 있으면 본문 뒤에 별도 섹션.
- 분량. 1-pager 800~1500자(영문 약 350~600 단어). 6-pager 4500~6000자(영문 1500~1800 단어). 본문 외 부록·데이터·FAQ는 분량 제한 밖.
- 금지. 본문에 불릿·번호 매김 사용. 본문에 표·차트 삽입(부록으로 분리). 결론 부재한 정보 나열. "검토가 필요"로 끝맺기. 한 단락에 결론 두 개. 회의 직전 배포(침묵 독해 시간 부재).

### 기획안, 제안서
- 문체. 한 줄 요약 → 배경 → 솔루션 → 실행 계획. 의사결정자 관점.
- 분량. 1500~3000자, A4 2~5페이지.
- 금지. 솔루션 없는 문제 나열. 실행 일정 부재. 비용 추정 부재.

### 영업 제안서, RFP
- 문체. 첫 페이지에 차별성 한 줄. 가격과 일정을 명확하게. 클라이언트 언어.
- 분량. 5~15페이지. 핵심은 첫 3페이지에.
- 금지. 자사 자랑이 차별성보다 길어짐. 모호한 가격 ("협의"). 일정에 버퍼 부재.

### 회의록
- 문체. 결정·할 일·보류의 세 칸. 시점·담당자 명시.
- 분량. 한 페이지, 500~1000자.
- 금지. 토의 내용을 그대로 옮기기. 결정 사항을 모호하게 적기. 담당자 부재.

### 이사회 보고서 — 9.7
- 문체. 결정 요청 한 줄을 머리에. 자신감과 겸손의 균형. 숫자는 출처와 함께.
- 분량. 안건당 1~2페이지, 1500~3000자. 첨부 자료 별도.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - 자기 변호.
  - "검토 중" 직접 인용 금지.
  - "선방" 직접 인용 금지.
  - 약속의 인플레이션.
  - 결정 요청 누락(보고서로 끝나는 안건).

### 주주 서한 — 9.7
- 문체. 결정 요청 한 줄을 머리에. 자신감과 겸손의 균형. 숫자는 출처와 함께.
- 분량. 2~5페이지, 3000~8000자. 분기 서한은 짧게, 연차 서한은 길게.
- 금지 (9.7군).
  - 자기 변호.
  - "견조한" 직접 인용 금지.
  - "전략적 우위" 직접 인용 금지.
  - 약속의 인플레이션.

### PRD
- 문체. 사용자 문제 한 줄, 솔루션 한 줄, 측정 지표 한 줄. P0/P1/P2 분류.
- 분량. 1500~5000자. 하나의 기능 단위면 1500~2500자, 큰 제품 단위면 3000~5000자.
- 금지. "좋은 UX", "더 편하게" 같은 추상 표현. 비범위(out of scope) 부재. 측정 방법 없는 성공 지표.

### IR·실적 발표 자료 — 9.7
- 문체. 첫 슬라이드에 한 줄 헤드라인 메시지(예. "QoQ 매출 +12%, 영업이익률 18% 유지"). 분기 비교는 표 한 장. 가이던스 별도 섹션.
- 분량. IR 자료 20~30장, 발화 스크립트 분당 130~150자(30분 기준 4000~4500자). 보도자료 동반은 600~1000자.
- 금지 (9.7군).
  - "견조한" 직접 인용 금지.
  - "안정적인" 직접 인용 금지.
  - "시장 상황을 면밀히 모니터링" 직접 인용 금지(가이던스 회피).
  - 출처 없는 비교 지표.
  - 형용사 가득한 자평.

### OKR·KPI 정기 리포트
- 문체. 한 페이지에 한 OKR. 목표·실측·격차 세 칸 표. 격차 원인 한 줄, 다음 액션 한 줄.
- 분량. 분기 OKR 리포트 페이지당 한 OKR, 전체 5~10페이지. 주간 KPI는 한 페이지 표 + 한 단락 코멘트.
- 금지. 빨강·초록·노랑 색만 표시하고 원인 부재. "노력하겠습니다" 류 액션 부재. 측정 기준 변경 무고지.

### MD&A · 재무 코멘터리
- 문체. 항목별 변동 원인 한 줄. 정량(YoY, QoQ) + 정성(원인) 균형. 가이던스 별도 문단.
- 분량. 분기 MD&A 3~8페이지, 연차 MD&A 8~20페이지.
- 금지. "견조한", "안정적인", "선방" 같은 형용사 자랑. 변동 원인 누락. 가이던스 회피.

### 투자 메모 (IC Memo) — 9.7
- 문체. 투자 결정 한 줄을 머리에. 비즈니스·시장·재무·리스크·exit 다섯 단. 가정 명시. 정량과 정성 균형.
- 분량. 5~15페이지, 7000~20000자. 사전 메모는 1~2페이지.
- 금지 (9.7군).
  - "유망" 직접 인용 금지.
  - "잠재력 있음" 직접 인용 금지.
  - "검증 필요" 직접 인용 금지.
  - exit 시나리오 부재.
  - 베이스·업·다운 케이스 부재.

### LP 리포트 (펀드 업데이트)
- 문체. 분기 메시지 한 줄을 머리에. 포트폴리오 별 한 단락(NAV·MOIC·IRR). 펀드 전반 요약.
- 분량. 분기 10~30페이지, 연차 50~100페이지.
- 금지. 자기 변호 일색. 비교 기준 변경 무고지. 손실 회피 표현. 약속의 인플레이션.

### 비즈니스 DD 메모 (Commercial DD)
- 문체. 가설을 한 줄로 시작. 검증 방법(인터뷰·서베이·데이터). 결과 → 시사점.
- 분량. 5~20페이지. 한 가설당 2~5페이지.
- 금지. 가설 없이 자료 나열. 인용 출처 부재. 정량과 정성의 분리 누락.

### 산업·시장 리포트 (Sector Note)
- 문체. 산업 변화의 인사이트 한 줄. 드라이버·승자·패자 셋. 데이터 출처와 함께.
- 분량. 5~15페이지. 차트 5~10개.
- 금지. 데이터 나열로 끝. 출처 누락. "추가 분석이 필요"로 끝맺기.

### 전문가 콜 노트 (Expert Call)
- 문체. 전문가 한 줄 소개. 질문·답변 쌍 형식. 핵심 인용 별도 박스. 후속 액션 한 줄.
- 분량. 한 콜당 2~5페이지. 60분 콜 기준 3000~5000자.
- 금지. 질문 없는 답변. 인용 부정확. 후속 액션 부재. 비공개 정보 유출.

### 스토리라인·결론 (Pyramid Principle)
- 문체. 결론 한 줄(서술형, 의문문 아님). 지지 결론 셋(MECE). 각 지지 결론에 사실 두세 개. So what 끝까지.
- 분량. 한 페이지. 본 덱 작성 전 단계 산출물.
- 금지. 결론가 질문형. 지지 결론 MECE 위반. 사실 없는 추론. 한 결론에 이슈 두 개.

### 진단 보고서 (Diagnostic Report)
- 문체. Executive Summary 한 페이지. 사실·해석·시사점 세 단. 데이터 출처 명시. 권고 별도 섹션.
- 분량. 20~50페이지. ES는 1~2페이지.
- 금지. 데이터 나열로 끝. 권고 부재. "추가 분석이 필요"로 끝맺기.

## fallback 5단계 합성 알고리즘

1. **결정자의 위치를 본다.** 임원 / 이사회 / 주주 / LP / IC / 외부 클라이언트 중 누구인가.
2. **결정의 종류를 본다.** 승인(go/no-go) / 지속(continue) / 자원 배분 / 평가 중 무엇인가.
3. **두 후보를 고른다.** 결정자 일치 + 결정 종류 일치가 1순위.
4. **문체 합성.** 첫 줄에 결정 요청 또는 메시지 + 근거 단락 + 가정 명시. 9.7 영역이면 BLACK 제약 두 가지 이상.
5. **금지 합성.** 형용사 자랑("견조한·안정적인") + 회피 어미("검토가 필요") + 약속 인플레이션 세 줄을 기본으로.

예. **"위원회 안건 보고서"**. 결정자 위원회. 결정 종류 승인. 후보 = 이사회 보고서 + 영업 제안서. 합성. 결정 요청 한 줄 + 차별성·근거 한 단락 + 일정·예산 명시. 9.7 영역이므로 자기 변호 + 모호한 "검토 중" 두 줄 직접 인용 금지.
