# 내부 커뮤니케이션 (회사 내부 수신자에게 보내는 글)

이 카테고리는 회사 안의 동료·팀·전사 직원에게 보내는 모든 글을 다룬다. 사내 공지·1on1·채용 공고·해고 통지·타운홀·HR 정책처럼 권한 차이와 신뢰 관계가 톤에 깊이 작용하는 영역이다. 외부 글과 다른 점은 (1) 수신자가 회사 정책·문화 맥락을 이미 안다, (2) 톤이 한 번 흔들리면 신뢰 회복에 분기 단위 시간이 든다, (3) 정치성·중립성이 늘 평가 축에 든다.

## 매핑 표

각 행은 RED=WHAT, SILVER=HOW, BLUE=FORM 카테고리에서 한 개씩 뽑혀 있다. 합격선이 표기되지 않으면 9.5다.

| 과제 | RED (WHAT) | SILVER (HOW) | BLUE (FORM) | 합격선 |
|------|-----|--------|------|--------|
| 사내 공지 | 결정 사항, 배경 | 톤, 공정성 | 분량, 명확성 | 9.5 |
| 1:1 피드백, 성과 리뷰 | 행동 변화 | 사례 구체성, 시점·장면 | 톤, 거리감 | 9.5 |
| 채용 공고 | JD 정합성, 매력 | 구조, 가독성 | 톤(지원자 끌기) | 9.5 |
| 어려운 메시지 (해고, 거절) | 결정의 명확성 | 후속 안내 구조 | 거리감·존중 톤 | **9.7** |
| 타운홀·전사 메시지 | 방향성, 결정의 무게 | 흐름: 현황 → 과제 → 실행 | 거리감 톤, 시간 분배 | 9.5 |
| 노조·HR 정책 발표 | 정책 변경의 근거 | 적용 범위·시점 구조 | 톤(공정성·중립성) | 9.5 |
| 정기 1on1 (운영) | 진행 안건 | 흐름: 지난 약속 → 현재 → 다음 | 거리감, 분량 | 9.5 |

## 합격선 기준 장면

- 사내 공지. "직원이 슬랙 한 번만 읽고 적용 시점을 정확히 안다."
- 1:1 피드백. "받는 사람이 다음 주 행동 한 가지를 메모해서 나간다."
- 채용 공고. "타깃 후보가 첫 화면에서 책임까지 읽고 지원 버튼을 누른다."
- 어려운 메시지 (9.7). "받는 사람이 한 번 더 묻지 않고 다음 절차로 넘어간다."
- 타운홀. "회의 끝나고 직원 셋이 같은 키워드 한 줄로 요약한다."
- 노조·HR 정책. "노조 측이 변경 근거를 인정하고 적용 범위만 협상 의제로 가져간다."
- 정기 1on1. "다음 1on1 직전에 지난 약속을 매니저와 직원 둘 다 기억한다."

## 문체·금지 뱅크

### 사내 공지
- 문체. 결정 한 줄, 배경 한 단락, 영향 한 줄, 적용 시점 한 줄.
- 분량. 300~800자. 슬랙 메시지 한 묶음으로 다 보이는 길이.
- 금지. "양해 부탁드립니다"식 회사 입장 회피. 모호한 "가급적".

### 1:1 피드백, 성과 리뷰
- 문체. 강점, 성장 영역, 한 가지 행동 변화의 세 단. 사례에 시점과 장면.
- 분량. 서면 피드백 500~1000자. 면담 노트는 한 페이지 내.
- 금지. "노력하세요" 류 추상 권고. 인격과 행동의 혼동. 추상 형용사 ("적극적인").

### 채용 공고
- 문체. 회사 한 줄, 역할 한 줄, 책임 다섯, 자격 다섯, 우대 셋.
- 분량. 800~1500자. 모바일 한 화면에서 책임까지 보여야 함.
- 금지. "열정", "주인의식" 같은 클리셰. 책임과 자격의 정합성 부재.

### 어려운 메시지 (해고, 거절) — 9.7
- 문체. 결정 한 줄을 머리에. 거리와 존중 사이의 톤. 후속 안내 한 줄.
- 분량. 200~500자. 한 화면에 다 들어오는 길이.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - 변명조의 길어진 도입.
  - "회사 사정상 어쩔 수 없이" 직접 인용 금지.
  - "더 좋은 기회가 있을 것입니다" 직접 인용 금지.
  - 후속 안내 부재.
  - 받는 사람의 입장에서 한 번 더 다듬지 않은 1차 초안 그대로 발송.

### 타운홀·전사 메시지
- 문체. 현황 한 단락, 과제 한 단락, 실행 한 단락의 세 단. 일인칭 단수 "저"로 시작 가능. 청중이 가져갈 한 가지를 마지막에.
- 분량. 발화 기준 10~20분, 1500~3000자. 슬라이드 동반 시 8~12장.
- 금지. 자랑 일색. "함께 만들어가요" 같은 수동적 마무리. 분기 숫자만 나열하고 행동 요청 부재.

### 노조·HR 정책 발표
- 문체. 정책 변경 한 줄, 근거 한 단락, 적용 범위·시점 한 단락, 문의 채널 한 줄. 중립적 어휘.
- 분량. 800~1500자. A4 한 장에 다 들어오는 길이.
- 금지. 회사 입장 일방 강조. 모호한 적용 범위. 문의 채널 부재. "직원 여러분의 이해를 부탁드립니다" 같은 일방 마무리.

### 정기 1on1 (운영)
- 문체. 지난 1on1 약속 점검 한 줄, 현재 안건 두 단락, 다음까지 할 일 한 줄. 의제 형식.
- 분량. 작성 의제 300~600자, 면담 30~45분.
- 금지. 평가성 어휘(1:1 피드백 자리). 잡담 의제. 행동 합의 부재.

## fallback 5단계 합성 알고리즘

1. **수신자를 본다.** 한 명(1on1형) / 한 팀(공지형) / 전사(타운홀형) / 외부 위원(노조·HR 정책형) 중 누구인가.
2. **권한 차이를 본다.** 수평(동료) / 위→아래(매니저→팀원) / 아래→위(팀원→매니저) / 양방향(전사 의견 수렴) 중 무엇인가.
3. **두 후보를 고른다.** 수신자 일치 + 권한 일치가 1순위.
4. **문체 합성.** 결정 한 줄 + 배경 + 적용 시점 구조를 기본 골격으로 두고, 권한 차이가 클수록 거리감 톤을 더 두드러지게.
5. **금지 합성.** 일방 회피 어미("양해 부탁드립니다") + 추상 권고("노력하세요") 두 줄을 기본 금지로 두고, 9.7 영역이면 흔한 실패 문구 직접 인용 두 줄 추가.

예. **"임원 코칭 노트"**. 수신자 의사결정자(임원). 권한 양방향(코치→임원이지만 임원이 결정권). 후보 = 1:1 피드백 + 보고서(실무, 1페이지). 합성. 강점 + 성장 영역 + 행동 변화 한 가지를 액션 타이틀 형태로. 금지. "노력하시면 좋겠습니다" + 인격과 행동 혼동.
