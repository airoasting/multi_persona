# 분석·법무

이 카테고리는 결함이 곧 손실 또는 분쟁으로 이어지는 모든 분석·법무 산출물을 다룬다. 엑셀 모델·시장 분석은 9.5군이지만, 계약서 검토·법률 의견서·송무 서면·감사 의견서·세무 의견서·실사 보고서는 모두 9.7군이다. 9.7군의 책임 무게는 추상이 아니다. 계약서 한 조항의 모호 표현이 수억 단위 손해배상으로 이어지고, 감사 의견서의 한정 사유 누락이 상장폐지 단초가 되며, 실사 보고서의 발견 사항 누락이 인수 후 분쟁으로 비화한다. 결함이 즉시 자본·신뢰·법적 책임으로 환산되는 영역이다. 공통점은 (1) 인용·출처가 본문의 신뢰를 결정한다, (2) 모호 표현 한 줄이 책임 분쟁의 단초가 된다, (3) 가정·전제를 명시 안 하면 의견 자체가 무효화된다.

## 분석 보고서의 결론 원칙

이 카테고리의 산출물은 **형식상 분석 보고서지만, 요약(Executive Summary)과 결론 섹션은 메모 톤으로 써야 한다.** 전략기획(시장·경쟁사·트렌드 분석, 캠페인 성과, 데이터 분석, 사내 진단 메모), 법무(법률 의견서, 계약서 검토 의견서), 회계·재무(감사 의견서, 세무 의견서, 재무 분석), M&A·감사(실사 보고서, 내부통제 평가)에 동일하게 적용된다.

- 형식 유지. 사실관계·쟁점·검토·결론의 정형 구조, 인용·출처·각주, 부록은 그대로 둔다.
- 메모 톤 정의. 결론 한 줄에 (a) 단정형 종결어미("~함", "~이다", "~권고"), (b) 행동 동사(진행·보류·수정·이행·거부·승인·재배분·축소·확대·정정), (c) 주체 명시(팀장·실장·본부장·임원, 또는 클라이언트·이사회·감사위원회·재무팀장) 셋이 동시에 들어간다. "사료된다", "보아야 할 것이다", "추가 검토가 필요", "고려할 만함", "긍정적으로 검토 가능"은 (a) 단정형 결여로 자동 위반.
- 메모 톤 예시 (부서별).
  - 전략기획. "Q3 캠페인은 ROAS 2.1로 목표 미달. 4분기는 신규 채널 비중을 30%로 축소하고 기존 채널 예산을 재배분할 것을 본부장께 건의함." (단정 + 행동 "축소·재배분" + 주체 "본부장")
  - 법무. "본 인수계약은 진술과 보증 조항 수정 후 이사회가 승인할 것을 권고함. 면책 한도 미수정 시 거래 보류 권고." (단정 + 행동 "수정·승인·보류" + 주체 "이사회")
  - 회계·재무. "당기 매출 인식 시점은 IFRS 15에 부합하지 않으며, 결산 마감 전 회계처리 정정과 외부감사 재검토를 재무팀장께 권고함." (단정 + 행동 "정정·재검토" + 주체 "재무팀장")
- 위치별 형식. 요약(Executive Summary)은 권고 한 줄 + 근거 3개 + 잔여 리스크 한 줄. 본문 결론 단락은 압축 재서술 + 단정 권고 한 줄. 전문가 영역(법무·회계·감사·세무·M&A)은 결론에 등급 라벨(High·Medium·Low) 또는 의견 라벨(적정·한정·부적정·의견거절) + 후속 조치 권고 필수.
- 이유. 의사결정자는 요약과 결론만 읽는다. 일반 부서 보고는 팀장·실장·본부장 선에서 90%가 결정되고, 임원 보고는 그 위 단계에서 결정된다. 전문가 영역은 CFO·법무팀장·감사위원회·이사회가 같은 패턴으로 읽는다. 본문은 검증·방어용 갑옷이고, 결론이 실제 의사결정을 만든다. 분석 보고서 형식 안에 메모의 결론력을 숨겨 넣는 것이 이 카테고리의 합격선이다.

## 매핑 표

각 행의 RED·SILVER·BLUE는 5-Color Harness 페르소나 정체성에 해당한다. RED 이성(논리·근거·결론 타당성), SILVER 분야 전문가(베테랑이 첫눈에 짚는 디테일), BLUE 공감(결정자의 행동 부담을 줄이는 신뢰 무게). 분석·법무 카테고리는 외부 수신자가 아닌 결정자(법무팀장·이사회·재무팀장·딜 매니저·재판부)가 BLUE의 대상이며, BLUE는 "그 결정자가 불안 없이 결론에 행동을 옮길 수 있는가"를 본다. 합격선이 표기되지 않으면 9.5다.

## 합격선 기준 장면

- 엑셀 모델. "다른 사람이 30분 안에 가정·계산·출력 흐름을 따라가고 셀 한 개를 바꿔도 결과가 깨지지 않는다."
- 시장 분석. "독자가 통찰 한 줄을 자기 의사결정에 그대로 옮긴다."
- 계약서 검토 (9.7). "사인 직전 한 조항에서 한 번 더 멈춰 묻고 싶어진다."
- 법률 의견서 (9.7). "클라이언트가 결론 한 줄을 보고 한 번 더 읽지 않고 의사결정한다."
- 소장·답변서 (9.7). "재판부가 첫 변론기일에 청구취지·청구원인을 한 번에 파악하고 보충 자료를 따로 청하지 않는다."
- 준비서면 (9.7). "재판부가 다음 기일에서 우리 측 쟁점 정리를 그대로 인용한다."
- 감사 의견서 (9.7). "감독 당국이 한정 사유 단락을 그대로 인용 가능하다."
- 세무 의견서 (9.7). "납세자가 결론 한 줄로 신고를 진행한다."
- 실사 보고서 (9.7). "ES 한 페이지만 보고 딜 진행 여부 1차 판단이 가능하다."
- 내부통제 평가. "감사위원회가 결함 표 한 장으로 우선순위를 정한다."

## 문체·금지 뱅크

### 엑셀 모델
- 문체. 가정 셀과 수식 셀을 색상으로 분리. 셀 코멘트로 출처 남기기. 한 시트 한 목적.
- 분량. 시트 5~10개. 가정 시트 1, 입력 시트 2~3, 계산 시트 2~4, 출력 시트 1~2.
- 금지. 가정 시트 밖의 하드코딩. 명명 없는 시트. 검증 시트 부재.

### 시장 분석
- 문체. 통찰 한 줄을 헤드에. 데이터 출처 명시. 비교군과 함께 보기.
- 분량. 3~10페이지, 5000~15000자. 차트 5~10개.
- 금지. 데이터 나열로 끝남. 출처 누락. 통찰 없는 요약.

### 계약서 검토. 9.7
- 문체. 짧고 단정적. 정의 조항으로 모호 표현 차단하기. 책임 한도와 면책의 양방향성 명시.
- 분량. 검토 의견서 2~5페이지, 3000~7000자. 조항별 의견은 한 조항당 100~300자.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - "고려할 수 있다" 직접 인용 금지.
  - "합리적 노력" 직접 인용 금지.
  - "통상의 관행" 직접 인용 금지.
  - 진술과 보증의 누락 묵과.
  - 책임 한도 일방성 묵과.

### 법률 의견서 (리걸 메모). 9.7
- 문체. 결론 한 줄을 머리에. 사실관계 한 단락. 법리 검토 본문. 결론 재서술. 각주에 출처. 가정·전제 명시.
- 분량. 5~15페이지, 5000~15000자. 짧은 사안은 2~3페이지.
- 금지 (9.7군).
  - "사료된다" 직접 인용 금지.
  - "보아야 할 것이다" 직접 인용 금지.
  - "해석상 다툼의 여지" 직접 인용 금지.
  - 가정 누락. 적용 시점 부재.
  - "추가 검토가 필요"로 끝맺기.

### 소장·답변서 (9.7군)
- 문체. 소장은 청구취지·청구원인 두 단락. 답변서는 청구취지에 대한 답변·청구원인에 대한 답변·항변 세 단락. 시계열·증거 일치. 당사자 표시 정확.
- 분량. 소장 10~30페이지. 답변서 5~20페이지. 단순 사안은 5~10페이지.
- 금지 (9.7군: 4~5개, 직접 인용 2개 이상).
  - "심히 부당" 직접 인용 금지.
  - "악의적" 직접 인용 금지.
  - 입증 없는 단정.
  - 형식 위반(송달 주소·당사자 표시 누락).
  - 청구취지가 모호한 추상 표현(특정 부재).

### 준비서면 (9.7군)
- 문체. 쟁점 정리 한 단락을 머리에. 상대방 주장 인용 → 우리 측 반박(법리·증거) → 소결의 3단을 쟁점마다 반복. 새 쟁점은 별도 단락 또는 별지.
- 분량. 15~50페이지. 단순 쟁점은 5~10페이지.
- 금지 (9.7군).
  - "사회 통념상" 직접 인용 금지.
  - "이미 법정에서 충분히" 직접 인용 금지.
  - 같은 주장을 다음 회차 준비서면에 재반복.
  - 상대방 주장 인용 부정확.
  - 쟁점 누락(상대 주장에 대한 무대응).

### 감사 의견서 (감사 보고서). 9.7
- 문체. 적정·한정·부적정·의견거절 중 하나 명시. 한정·부적정이면 사유 단락 필수. 회계기준 인용. 비교 시점 일치.
- 분량. 본문 1~2페이지(의견서), 부속서 별도. 핵심감사사항(KAM)은 항목당 300~500자.
- 금지 (9.7군).
  - "전반적으로 적절" 직접 인용 금지.
  - "특별한 문제 없음" 직접 인용 금지.
  - 한정 사유 부재.
  - 비교 시점 불일치.
  - 회계기준 인용 누락.

### 세무 의견서 (세무 메모). 9.7
- 문체. 결론 한 줄 머리에. 세법·예규·판례 인용 본문. 리스크 등급(낮음·중간·높음) 표시. 결론 재서술.
- 분량. 3~10페이지, 3000~10000자.
- 금지 (9.7군).
  - "유사 사례 기준" 직접 인용 금지.
  - "관행에 따라" 직접 인용 금지.
  - "통상" 직접 인용 금지.
  - 시행령·시행규칙 출처 누락.
  - 적용 시점·납세자 식별 부재.

### 실사 보고서 (Due Diligence). 9.7
- 문체. Executive Summary 한 페이지를 머리에. 카테고리별 발견 사항(재무·법무·운영·세무·인사). 리스크 우선순위 표(High·Medium·Low).
- 분량. 30~100페이지. ES는 1~2페이지. 각 카테고리 5~15페이지.
- 금지 (9.7군).
  - "검토가 필요해 보임" 직접 인용 금지.
  - "향후 확인" 직접 인용 금지.
  - 출처 없는 발견.
  - 리스크 등급 부재.
  - 후속 조치 권고 부재.

### 내부통제 평가 보고서 (Management Letter)
- 문체. 결함 한 줄, 영향 한 줄, 권고 한 줄의 세 칸 표. 우선순위 등급. 책임자·기한 명시.
- 분량. 5~20페이지. 결함 항목당 200~400자.
- 금지. 등급 부재. 권고가 모호("개선 필요"). 책임자·기한 누락.

## fallback 5단계 합성 알고리즘

1. **결과물의 책임 무게를 본다.** 의견(클라이언트 의사결정 근거) / 보고(외부 공시) / 검토(내부 참고) / 모델(반복 사용) 중 무엇인가.
2. **인용 강도를 본다.** 법령·기준 직접 인용 필수 / 출처 표기만 / 자체 분석 중 무엇인가.
3. **두 후보를 고른다.** 책임 무게 일치 + 인용 강도 일치가 1순위.
4. **문체 합성.** 결론 한 줄 + 사실·법리·결론 또는 가정·계산·출력의 3단 + 출처 명시.
5. **금지 합성.** 모호 표현("고려할 수 있다·합리적 노력") + 회피 어미("추가 검토가 필요") 두 줄을 기본 금지로. 9.7군이면 그 영역 흔한 실패 문구 직접 인용 두 줄 추가.

예. **"보안 심사 보고서"**. 책임 무게 의견(공식 심사 결과). 인용 강도 기준(ISO·NIST) 직접 인용. 후보 = 감사 의견서 + 내부통제 평가 보고서. 합성. 적합·부적합 결론 한 줄 + 통제 항목별 결함·영향·권고 표 + ISO/NIST 인용. 9.7군 처리. "전반적으로 양호" 직접 인용 금지 + 등급 부재 금지.
